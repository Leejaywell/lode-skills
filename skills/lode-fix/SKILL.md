# Bug Fixer（修 Bug）

扩展技能。系统化修复缺陷：先复现、再定位、做最小修复、回归验证。

## Usage（什么时候用）

- 自测或测试失败、行为不符合预期。
- `lode-review` 打回的阻塞项。
- 线上/运行期异常需要排查修复。

## Done（什么算合格）

- 先**稳定复现**问题（写一个能复现的失败用例最佳）。
- 定位**根因**而非表象，说清为什么会错。
- 做**最小修复**，不顺手重构无关代码。
- 回归：原失败用例转绿，且既有测试全过。
- 若这是一次值得记住的真实踩坑 → 提示 `/lode-evolve` 把它沉淀成规则。

## Guardrails（红线）

- 不靠猜：没复现就不算定位，别盲改一通。
- 修实现，别改测试来掩盖问题（除非测试本身写错）。
- 一次修一个根因，别夹带无关改动扩大影响面。
- 改完跑全量相关测试，确认没引入新红。
