# Drive（自主驱动器）

Lodestar 的自主大脑。把"一个目标 → 自主跑完 → 新老通吃"落地的就是它：它不是写一条 Go，而是**驱动整条主线循环**，靠一个能续命的**进度账本**跑到底。

> 自主 ≠ 无人。它自己驱动全程，人只在两处出现：**审 PR** 和 **接熔断**。

## Usage（什么时候用）

- 用户给一个目标，要 agent **自主跑完整个目标**，不想一步步盯。
- 目标可能跨多个 Face / 多个 Session，需要崩了能续、跑完能审计。

## 开局先定两个模式（决定走多重的护栏）

1. **绿地 ↔ 棕地**：有现成代码就是棕地 → 先 `lode-recon` 出 `system-map.md`，spec 走 delta 模式，plan 做影响分析，verify 跑**全量回归**。绿地走精简流程。
2. **单人 ↔ 团队**：单人用本地 `review-passed` 门禁；团队/长生命周期切 **PR/CI 门禁**——完成 = PR 过 CI + 必需 approval 合并。

## 怎么跑（驱动循环）

1. **定目标**：把目标写进 `.lode/<project>/goal.md`（目标 + 可验收的完成标准）。
2. **拆解**：目标 → 里程碑 → 有序 Face（每个 Face 一条 Goal，标依赖/可并行/爆炸半径）。写进 `dev-plan.md`。
3. **建账本**：`.lode/<project>/ledger.jsonl`，每个 Face 一条记录（状态 + commit/PR + 时间）。
4. **循环**：读账本 → 挑下一个**无阻塞** Face → 用 `lode-build` 做 → **四步审计 + 全量回归** → 提交（团队模式开 PR 等 CI/审）→ **更新账本** → 下一个。
5. **重规划**：某个 Face 暴露出计划错了 → 回 `lode-plan` 改计划再继续，**别照着过时计划硬跑**。
6. **熔断**：同一 Face 连续失败 ≥3 次、或超出 token 预算 → 停下找用户，把卡点和已知信息一次讲清。
7. **收尾**：所有里程碑达成 → 跑 `lode-release`（或合并最后 PR）→ 对照 goal.md 完成标准自检。

## Done（什么算合格）

- `goal.md` + `dev-plan.md` + `ledger.jsonl` 三件齐全且持续更新；任何时刻能从账本看出"做到哪、还差什么"。
- 每个 Face 都过了四步审计 + 回归门禁（团队模式则 PR 合并），账本如实记录。
- 目标达成 = `goal.md` 的完成标准逐条满足，用证据（命令输出/PR/审查）说话。
- 中途崩溃可从账本无损续跑；全程可审计。

## Guardrails（红线）

- **账本是真相**：状态只在四步审计/回归/PR 真过之后才写 passed，不许提前乐观标记。
- **熔断优先于硬跑**：门禁挡"坏的完成"，熔断挡"昂贵的不完成"；卡住就停，别烧 token。
- 棕地必须先 `lode-recon`、必须跑全量回归；**没有基线就不许动老代码**。
- 团队模式下"完成"= 合并，不是本地标记；子代理审查只是 PR 前过滤，不替代人审。
- 一个目标一个账本；并行 Face 不碰同一文件，冲突由主 Agent 合并。
- 决策权始终在人：对外不可逆动作（push 已被 PR 流程覆盖；上架/部署）执行前确认。
